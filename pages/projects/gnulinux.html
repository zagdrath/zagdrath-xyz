What is GNU/Linux?

Linux or GNU/Linux is a family of open-source Unix-like operating systems based on the Linux kernel, an operating system kernel first released on September 17, 1991, by Linus Torvalds and The GNU Project, started in 1983 by Richard Stallman who had the goal of creating a complete Unix-compatible software system composed entirely of free software. GNU/Linux at its core is just the Linux kernel along with the GNU system and it makes up a complete operating system.

Together, releases of GNU/Linux are often dubbed distributions or distros, which include the Linux kernel and supporting system software and libraries, many of which are provided by the GNU Project. Commonly mentioned GNU/Linux distributions include Debian, Ubuntu, Fedora, and Arch. Desktop GNU/Linux distributions include a windowing system such as X11 or Wayland, and a desktop environment such as GNOME or KDE. Because GNU/Linux is freely re-distributed, anyone may create a distribution for any purpose, and you too may be compelled to do so.

The Philosophy of GNU/Linux and why use GNU/Linux

The basis of most of the following philosophies, however, is the Free Libre Software Philosophy, FLOSS, or the GNU Philosophy, which is a focus of an open and shared operating systems which is not only free as in money, often dubbed as "Gratis", but free as in freedom, dubbed as "Libre". The GNU/Linux operating systems gives users more control over their computing experience than competing operating systems such as Windows, which could contain malicious features without the user's knowledge. Picking a Distribution

Picking your first GNU/Linux distribution can be challenging for those who may not be too sure what they are actually looking for. In this section I will be giving a brief introduction to the main branches of GNU/Linux and the steps to take to pick your own Linux distribution that meets your needs. Often when looking into the history of GNU/Linux distributions they are often shown something like a large tree forking into smaller and smaller branches. This is often overwhelming to new and old users alike due to the sheer scope and branches GNU/Linux has had over it's generations. To simplify this tree I will be breaking it down to the main branches in which a majority of GNU/Linux distributions are forked which is shown below.

~> Slackware (Released in 1993)

Slackware was developed by Patrick Volkerding and was designed with two main objectives in mind, ease of use and stability. Originally based on Softlanding Linux System, Slackware has been the basis for many other distributions, most notably the first versions of SUSE Linux distributions, and is the oldest distribution that is still maintained. Slackware was also designed to be the most "Unix-like" GNU/Linux distribution. This is considered the first in a great legacy of GNU/Linux operating systems.

~> Debian (Released in 1993)

Debian was developed by the community-supported Debian Project, which was established by Ian Murdock. Debian is also the basis for many other distributions, most notably Ubuntu. Debian is one of the oldest operating systems based on the Linux kernel. Debian comes with over 50,000 packages, compiled in their repositories with thousands of programmers supporting the operating system.

~> Linux From Scratch (Released in 1999)

Linux From Scratch is a project that provides you with the steps necessary to build your own custom GNU/Linux system, entirely from source code. This is also a great way to learn how different aspects of GNU/Linux work. It also is a great way to have a GNU/Linux distro very customized to your own taste and needs.

~> Gentoo (Released in 2000)

Gentoo is a GNU/Linux distribution initially developed by Daniel Robbins, and it is built using the Portage package management system. Unlike other distribution's package managers, Portage is a true ports system in the tradition of BSD ports, but is Python based and sports a number of additional features.

~> Arch (Released in 2002)

Arch is an independently developed distribution targeted at competent GNU/Linux users who want to customize and build their own distribution but is unwilling to use LFS or Gentoo. It uses 'pacman', its home-grown package manager, to provide updates to the latest software applications with full dependency tracking. The default install provides a solid base that enables users to create a custom installation.

~> Fedora (Released in 2003)

Fedora is a GNU/Linux distribution developed by the community and owned by Red Hat. Fedora contains software distributed under a free and open source license. Fedora is focused on innovation over the stability of RHEL, integrating new technologies early on and working closely with upstream GNU/Linux communities. Fedora also offers spins which have specific sets of software, offering alternative desktop environments or specific interests such as gaming, security, design, scientific computing and robotics.

These are the foundation of most GNU/Linux distributions and is what I recommend for people to look at for distros or any forks of these distros. I recommend these because most of them will give you a long lifespan, more stability, and more support as a user when troubleshooting issues. What is Systemd and why Does it Matter?

Systemd is a software suite that provides a collection of system components for the GNU/Linux operating system. The primary purpose of Systemd is to script and organize boot functions for GNU/Linux using a system and service manager, which is an init system. An init is an initialization program, which is the first process started during booting of the computer system. The init process is directly started by the kernel, in this case, the Linux kernel, and will continue running until the computer is shut down. Systemd used this init system to bootstrap user space and manage user processes. It also provides replacements for various daemons and utilities, including device management, login management, network connection management, and event logging.

What is a Desktop Enviroment and Which one Should you Pick?

A desktop environment is a graphical user interface or GUI where you are able to interface with the entire computer graphically rather than just through a terminal. A desktop environment also encompasses a lot of other tools to assist the user in graphical computing. Furthermore a desktop environment utilizes a widget toolkit, which provides a set of controls that display information to the user. Currently there are two big tool-kits, GTK (GIMP ToolKit) and Qt. Below is a brief breakdown of the most popular and commonly used desktop environments.

~> Xfce (Released in 1996)

Xfce is a lightweight desktop environment that aims to be fast and low on system resources, while still being visually appealing and user friendly. It consists of separately packaged parts that together provide all functions of the desktop environment, but can be selected in subsets to suit user needs and preference. Xfce uses the GTK graphical tool kit and is licensed under the GPL, LGPL, and BSD licenses.

~> Enlightenment (Released in 1996)

Enlightenment is a window manager, compositor and minimal desktop and the project is currently primarily supporting X11 but has experimental Wayland support. Enlightenment includes functions to provide a graphical shell, and it can be used in conjunction with programs written for GNOME or KDE. Enlightenment uses the EFL graphical tool kit and is licensed under the BSD license.

~> KDE Plasma (Released in 1998)

KDE Plasma (often called just KDE) is a popular desktop environment for the X Window System. The graphical interface uses openGL for hardware acceleration, which results in better performance and reduced power consumption. KDE Plasma uses the Qt graphical tool kit and is licensed under the LGPL license.

~> GNOME (Released in 1999)

GNOME is a desktop environment that aims to be simple and easy to use. It is designed by The GNOME Project and is composed entirely of free and open-source software. GNOME 3 is the default desktop environment on many major GNU/Linux distributions. GNOME uses the GTK graphical tool kit and is licensed under the GPL and LGPL licenses.

~> Cinnamon (Released in 2011)

Cinnamon is a free and open-source desktop environment for the X Window System that derives from GNOME 3 but follows traditional desktop metaphor conventions. As the distinguishing factor of Linux Mint, Cinnamon has generally received favorable coverage by the press, in particular for its ease of use and gentle learning curve. With respect to its conservative design model, Cinnamon is similar to the Xfce and GNOME 2 desktop environments. Cinnamon uses the GTK graphical tool kit and is licensed under the GPL license.

~> MATE (Released in 2011)

The MATE Desktop Environment is the continuation of GNOME 2. It provides an intuitive and attractive desktop environment using traditional metaphors for GNU/Linux and other Unix-like operating systems. MATE uses the GTK graphical tool kit and is licensed under the GPL and LGPL licenses.

~> LXQt (Released in 2014)

LXQt is a free and open source lightweight desktop environment. It was formed from the merger of the LXDE and Razor-qt projects. LXQt is a very lightweight desktop environment and it uses the Qt graphical tool kit and is licensed under the GPL and LGPL licenses.

Starting with Enlightenment, XFCE, and LXQt being in the lower end of the specs, at or around 100 MiB of usage. These desktop environments boot up very snappy and range from 5-10 seconds from boot to terminal. These lower spec Desktop Environments also have a common focus on a Mouse Centered work-flow. In this category based on resources, ease of use, and aesthetics, I would recommend XFCE to start. If you need to keep the looks of a proper desktop but also desire to squeak out a bit more power from your system, Enlightenment is a great option. For those moving from Windows and need a familiar space I would look into using LXQt.

Following with MATE, Cinnamon, GNOME, and KDE these are our heavier usage desktop environments, these often show RAM spikes at about 150 - 300 MiB. Boot times vary, ranging from 20-45 seconds to terminal. The trend with these desktop environments as well, is that they follow a more Keyboard Centered design for easier work-flow and Touch-Screen functionality. All are clean and look like a modern desktop. I would recommend either KDE if you are coming from Windows and need a familiar space. For those wanting to feel what modern GNU/Linux is like, I recommend GNOME or personally I prefer Cinnamon.

Window Managers

Currently most GNU/Linux and BSD distributions use the X Window System for drawing the desktop, while the X System does not manage the windows, instead it depends on a special client application called a window manager. The window manager moves and sizes windows in response to user input. Window managers can be divided into these three categories. Stacking, which allows windows to draw their contents one on top of another on the desktop, starting with the one on the bottom and going up in the "Z order". Compositing , provides a buffer for each window to draw on and then compose those buffers together creating the desktop image. This type of window manager allows use of semitransparent windows. Tiling, in which the windows do not overlap. Tiling Window Managers can also be divided into Static tiling WMs and Dynamic tiling WMs.
